> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) 문법으로 추적한다.

**Goal:** ci-hands-on 저장소에 노트 03의 6계층 게이트 + provenance + 노트 02의 self-healing CI를 PR 7개로 실제 적용한다.

**Architecture:** 기존 `ci.yml`의 집계 게이트(`ci-success`) 패턴에 새 잡을 편입한다. CodeQL과 Claude 에이전트 2종만 별도 워크플로. 각 PR은 "게이트 추가(초록) → 위반 커밋(빨강) → 복구(초록) → 머지" 사이클로 검증한다.

**Tech Stack:** GitHub Actions, gitleaks-action@v2, CodeQL(codeql-action@v3), dependency-review-action@v4, pip-audit, mutmut 3, attest-build-provenance@v3, anthropics/claude-code-action@v1

## Global Constraints

- 저장소: `zzzonghwa/ci-hands-on` (public, 로컬 경로 `~/learning/cicd/hands-on`)
- 브랜치 보호: main, 필수 체크 현재 `ci-success` 1개 (Task 2에서 code scanning 체크 추가)
- 커밋 메시지: Conventional Commits, 한글 subject 30자 이내, AI attribution 금지
- 브랜치명: `feature/<short-name>` (이슈 없음)
- 머지: squash + 브랜치 삭제. **머지 직전 반드시 사용자 확인** (초록/빨강 증거 링크 제시 후)
- 워크플로 주석: 기존 스타일 유지 — 각 단계에 "왜"를 한글로
- 빨간불 시연 커밋은 복구(reset/revert) 후 머지. 시크릿 시연은 히스토리에서 제거(force push)까지
- 모든 push 후 확인 명령: `gh pr checks <PR번호> --watch` (또는 `gh run watch`)

    return path

    """(시연용) 입력 문자열을 그대로 실행 — 코드 인젝션."""

```python
"""mutmut run 출력에서 mutation score를 계산해 80% 미만이면 실패시키는 게이트.

사용: python mutation_gate.py mutmut.log
mutmut 3의 진행 표시줄 집계(이모지 카운트)를 파싱한다:
🎉 killed / 🫥 no tests / ⏰ timeout / 🤔 suspicious / 🙁 survived / 🔇 skipped
"""

import re
import sys

THRESHOLD = 80.0

log = open(sys.argv[1], encoding="utf-8").read()
counts = re.findall(
    r"🎉 (\d+).*?🫥 (\d+).*?⏰ (\d+).*?🤔 (\d+).*?🙁 (\d+).*?🔇 (\d+)", log
)
if not counts:
    sys.exit("mutmut 집계 라인을 찾지 못함 — 출력 형식을 확인하라")

killed, no_tests, timeout, suspicious, survived, skipped = map(int, counts[-1])
denom = killed + no_tests + timeout + suspicious + survived  # skip만 분모 제외
score = 100.0 * killed / denom if denom else 0.0
print(f"mutation score: {score:.1f}% (killed {killed} / {denom})")
if score < THRESHOLD:
    sys.exit(f"게이트 실패: score {score:.1f}% < {THRESHOLD}%")
print("게이트 통과")
```

로컬 검증: `mutmut run | tee mutmut.log && python mutation_gate.py mutmut.log` → score 출력 확인. **survived가 있어 80% 미만이면** 먼저 테스트를 보강해 80% 이상으로 만든 뒤 진행한다(그 자체가 계층 4의 목적).

## 실행 중 공통 판단 규칙

- If 액션 버전이 존재하지 않는다는 에러 (`@v3` 등) → 해당 액션 저장소의 최신 메이저 태그 확인 후 교체, 계획 문서에 기록.
- If 시연용 위반이 예상 게이트에 안 걸림 → 게이트 설정(쿼리 스위트·룰)을 먼저 의심하고 조정. 위반 코드를 더 노골적으로 바꾸는 건 그 다음.
- If 에이전트 워크플로(claude-*)가 인증 실패 → `gh secret list`로 시크릿 존재 확인 → 사용자에게 `! claude setup-token` 재실행 요청.
- If 한 태스크에서 연속 3회 같은 실패 → 중단하고 사용자와 상의 (rabbit hole 방지).
